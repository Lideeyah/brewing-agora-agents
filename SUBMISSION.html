Project Description

The Problem

AI agents are getting capable enough to hire other agents for specialized sub-tasks — but there's no trust layer for agent-to-agent payments. An employer agent has no way to guarantee a worker agent actually delivers before paying, and a worker agent has no guarantee the employer won't ghost after the work is done. Human payment rails (Stripe, bank transfers) require identity and don't work at $0.10 per task.

The Solution

Brewing is a trustless escrow protocol for AI agent economies. An employer agent posts a job and locks USDC in a smart contract. A worker agent (Claude) completes the task autonomously. When the employer approves, USDC is released instantly. If the worker misses the SLA deadline, the contract slashes the job and refunds the employer — all without any human in the loop.

Why Arc

Arc L1 is Circle's EVM chain where USDC is the native gas token — like ETH on Ethereum, but stable. An agent pays for a job with the same asset it earns from jobs. No wrapping, no bridging, no ERC20 approve transactions. A $0.10 task costs ~$0.01 in fees with ~400ms finality. This makes per-task micropayments economically viable in a way that's impossible on chains with volatile gas tokens.

Technical Highlights

The core contract is AgentEscrow.vy — 147 lines of Vyper 0.4.0 using msg.value/send() for native USDC. State is always updated before transfers (no reentrancy vector). A FastAPI backend wraps the Arc RPC with async rate limiting. Claude claude-opus-4-5 runs in an autonomous loop: it receives job specs, completes research/analysis/strategy tasks, and the employer settles via smart contract. 22 titanoboa tests cover the full lifecycle in 0.38s.

What's Next

IPFS-linked job specs (already in the contract as ipfs_spec_hash), multi-agent task graphs where agents decompose and delegate sub-tasks, and on-chain reputation scores derived from SLA breach history. The contract is ready for mainnet with the same gas-token economics.

Judge Setup (5 minutes)

# 1. Clone
git clone <repo> && cd arc

# 2. Install (Python 3.11 required)
python3.11 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# 3. Configure (use our live contract — no deploy needed)
cp .env.example .env
# Set ARC_RPC_URL, ARC_PRIVATE_KEY, ANTHROPIC_API_KEY
